# Context Continuity Skill

## 功能

当用户表达想要"新建对话"或"继续"时：
1. 读取 MEMORY.md 提取长期记忆
2. 读取当天 memory/YYYY-MM-DD.md 提取最近上下文
3. 总结当前进行中的任务、项目、待办事项
4. 将上下文以简洁的摘要形式呈现给用户

## 触发条件

用户说以下话时触发：
- "新建对话"
- "继续"
- "接着上次"
- "上次我们做什么了"
- "继续之前的工作"
- "回顾上下文"
- "帮我记住"

## 执行步骤

### 1. 检查记忆文件

首先检查以下文件是否存在：
- `MEMORY.md` - 长期记忆
- `memory/YYYY-MM-DD.md` - 当天记忆
- `memory/YYYY-MM-DD.md` - 昨天记忆（备用）

```bash
# 读取长期记忆
cat MEMORY.md

# 读取今天的记忆
cat memory/$(date +%Y-%m-%d).md
```

### 2. 提取关键信息

从记忆中找出：
- 当前进行的任务/项目
- 未完成的事项
- 重要决策或进展
- 用户偏好或设置
- 任何标记为重要的信息

### 3. 生成上下文摘要

用 3-5 句话总结当前状态，例如：
> "我们正在开发一个 PDF 处理技能，已完成基础框架。目前遇到的问题是 XXX。上次你让我..."

### 4. 呈现给用户

将摘要呈现后，询问用户是否要基于这个上下文继续。

## 主动记录功能

当完成重要任务时，可以主动询问用户是否需要记录：

> "需要我记录到记忆里吗？下次可以续接上下文。"

记录内容包括：
- 完成的任务
- 重要决策
- 未完成的事项
- 用户偏好

## 注意事项

- 只提取与当前任务相关的记忆，跳过无关的历史
- 保持摘要简洁，不超过 200 字
- 如果没有记忆，告知用户"这是新会话，没有历史上下文"
- 尊重用户隐私，不要在摘要中暴露敏感信息
- 每次重要对话结束后，建议询问是否需要记录

## 文件位置

- 长期记忆：`MEMORY.md`
- 日常记忆：`memory/YYYY-MM-DD.md`
- 工作空间：`~/.openclaw/workspace/` 或当前工作目录
